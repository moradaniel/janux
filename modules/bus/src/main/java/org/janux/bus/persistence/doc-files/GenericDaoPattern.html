Generic DAO Pattern

Description
The Generic DAO Pattern is useful to avoid duplicating CRUD operations in every DAO of our domain model. Using Java5 generics we can define common behavior using parameterized methods. Each parameter is a generic class. We can extend any DAO from this pattern, the actual class is resolved in runtime. In the diagram we can see the following components:
GenericDaoReadOnly: This interface only contains read operations for searching and loading objects. A read only DAO would implement this interface.
GenericDaoWrite: This interface only contains write operations. A full-featured DAO woould implement both GenericDaoReadOnly and GenericDaoWrite.
GenericDaoHibImpl: Full-featured base class for all DAOs implemented using Hibernate and the Generic DAO pattern. Subclasses the Spring class HibernateDaoSupport to get Springs support for handling the Hibernate Sessionfactory and Sessions.
PersonDao: This interface specifies non CRUD operations specific for the Person entity.
PersonDaoImpl: implements the PersonDao interface and extends the abstract class GenericDaoHibImpl. Instantiates the parameterized type T with the class PersonImpl.
